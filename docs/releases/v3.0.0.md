# Process Monitor & Manager — v3.0.0

> *Codename: Pulse*

---

## 📖 Overview

Pulse is the first feature release built on top of the verified Bedrock baseline — it adds search, threshold-based alerts, a live auto-refreshing dashboard, and colorized CLI output, and brings the repository up to a standard open-source project layout: changelog, release docs, contribution guide, issue templates, CI, and developer tooling.

Building on the trustworthy foundation established in **Bedrock**, this release focuses on making the tool genuinely useful day-to-day — spotting a runaway process shouldn't require scrolling a static list — while also making the project itself easy for someone else to clone, understand, and contribute to.

---

## 🎯 Objectives

This milestone was developed with the following goals:

* Let a user find a specific process without scrolling the full list.
* Surface high-load processes without re-running the full listing repeatedly.
* Provide a live, auto-refreshing view for ongoing monitoring instead of one-shot snapshots.
* Bring the repository up to a standard open-source structure — docs, CI, issue templates, and contributor tooling.

## ✨ Added

### 🔎 Search Process

Filter the process table by name or PID — CLI option 6, or the GUI's "Search a Process" action.

---

### 🚨 High CPU/MEM Alerts

A one-shot report of every process currently at or above the 50% CPU/MEM threshold — CLI option 7, or GUI "Show Alerts".

---

### 📈 Live Dashboard

`modules/dashboard.py` — a curses-based, auto-refreshing terminal UI, built around two classes:
- **`ProcessManager`** — fetches and parses `ps` output, filters by query, and sends `kill`/`STOP`/`CONT` signals.
- **`Dashboard`** — owns the render/input loop: auto-refresh every 2 seconds, in-place search (`/`), and kill/suspend/resume (`k`/`s`/`r`) without leaving the view.

Launched from CLI option 8 or the GUI's "Live Dashboard" action (which opens it in a detected terminal emulator).

---

### 🎨 Colorized CLI Output

Process listings are now color-coded by CPU load: red at ≥50%, yellow at ≥20%, green below.

`assets/logo.svg` — a terminal-window mark whose traffic-light dots echo the tool's own kill/suspend/resume actions, with a live-pulse waveform for the monitoring theme. Shown at the top of the README.

## 🐛 Fixed

### 🧩 Dashboard Multi-Word Command Parsing

`modules/dashboard.py`'s process-line parser previously split on a fixed token count, which misaligned the `%MEM`/`%CPU` columns for multi-word commands (e.g. `python3 -m http.server 8000`). It now takes PID/PPID as the first two tokens and `%MEM`/`%CPU` as the last two, treating everything in between as the command.

---

## 🏛️ Architecture Progress

**Bash core, Python for stateful UI.** Pulse introduces the project's first non-Bash component: `modules/dashboard.py`. Simple one-shot queries (info, search, alerts) stay in Bash, matching Spark and Bedrock; the live, stateful, key-driven dashboard is implemented in Python with `ProcessManager` and `Dashboard` classes, since a real render/input loop is significantly harder to reason about in Bash than in a language with actual objects and a `curses` standard library.

**Documentation as a first-class citizen.** The `docs/` hierarchy, issue templates, and CI move the project from "a script someone can read" to "a repository someone can contribute to" — the same arc Bedrock completed for runtime correctness, Pulse completes for maintainability.

Known limitations at this release:

* No configurable alert/color thresholds — the 50%/20% cutoffs are constants in [modules/monitor_module.sh](../../modules/monitor_module.sh) (see [ROADMAP.md](../../ROADMAP.md)).
* No automated unit-test framework — `tests/smoke_test.sh` covers functional behavior, not isolated unit coverage.
* GUI's "Live Dashboard" action depends on a detected terminal emulator being installed (see [Troubleshooting](../troubleshooting/Troubleshooting.md)).

## 🏁 Release Summary

With search, alerts, a live dashboard, colorized output, and a full open-source repository structure, Pulse completes the arc started with Spark's initial scaffold and Bedrock's verified baseline. No version is currently in development. See [ROADMAP.md](../../ROADMAP.md) for planned directions — configurable alert thresholds, interactive dashboard sorting, and CSV export — and [CONTRIBUTING.md](../../CONTRIBUTING.md) to get involved.
