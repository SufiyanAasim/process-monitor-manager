# Process Monitor & Manager тАФ v3.0.0

> *Codename: Pulse*

## ЁЯУЦ Overview

Pulse is the first feature release built on top of the verified Bedrock baseline тАФ it adds search, threshold-based alerts, a live auto-refreshing dashboard, and colorized CLI output, and brings the repository up to a standard open-source project layout: changelog, release docs, contribution guide, issue templates, CI, and developer tooling.

Building on the trustworthy foundation established in **Bedrock**, this release focuses on making the tool genuinely useful day-to-day тАФ spotting a runaway process shouldn't require scrolling a static list тАФ while also making the project itself easy for someone else to clone, understand, and contribute to.

---

## ЁЯОп Objectives

This milestone was developed with the following goals:

* Let a user find a specific process without scrolling the full list.
* Surface high-load processes without re-running the full listing repeatedly.
* Provide a live, auto-refreshing view for ongoing monitoring instead of one-shot snapshots.
* Bring the repository up to a standard open-source structure тАФ docs, CI, issue templates, and contributor tooling.

### ЁЯЪи High CPU/MEM Alerts

### ЁЯУИ Live Dashboard

`modules/dashboard.py` тАФ a curses-based, auto-refreshing terminal UI, built around two classes:
- **`ProcessManager`** тАФ fetches and parses `ps` output, filters by query, and sends `kill`/`STOP`/`CONT` signals.
- **`Dashboard`** тАФ owns the render/input loop: auto-refresh every 2 seconds, in-place search (`/`), and kill/suspend/resume (`k`/`s`/`r`) without leaving the view.

Launched from CLI option 8 or the GUI's "Live Dashboard" action (which opens it in a detected terminal emulator).

## ЁЯРЫ Fixed

### ЁЯзй Dashboard Multi-Word Command Parsing

`modules/dashboard.py`'s process-line parser previously split on a fixed token count, which misaligned the `%MEM`/`%CPU` columns for multi-word commands (e.g. `python3 -m http.server 8000`). It now takes PID/PPID as the first two tokens and `%MEM`/`%CPU` as the last two, treating everything in between as the command.

## ЁЯПЫя╕П Architecture Progress

**Bash core, Python for stateful UI.** Pulse introduces the project's first non-Bash component: `modules/dashboard.py`. Simple one-shot queries (info, search, alerts) stay in Bash, matching Spark and Bedrock; the live, stateful, key-driven dashboard is implemented in Python with `ProcessManager` and `Dashboard` classes, since a real render/input loop is significantly harder to reason about in Bash than in a language with actual objects and a `curses` standard library.

**Documentation as a first-class citizen.** The `docs/` hierarchy, issue templates, and CI move the project from "a script someone can read" to "a repository someone can contribute to" тАФ the same arc Bedrock completed for runtime correctness, Pulse completes for maintainability.

Known limitations at this release:

* No configurable alert/color thresholds тАФ the 50%/20% cutoffs are constants in [modules/monitor_module.sh](../../modules/monitor_module.sh) (see [ROADMAP.md](../../ROADMAP.md)).
* No automated unit-test framework тАФ `tests/smoke_test.sh` covers functional behavior, not isolated unit coverage.
* GUI's "Live Dashboard" action depends on a detected terminal emulator being installed (see [Troubleshooting](../troubleshooting/Troubleshooting.md)).
